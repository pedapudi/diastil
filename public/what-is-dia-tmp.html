the premise · 02

The file is the format.

A deck is a closed profile of HTML — small enough for a validator to hold forever.

grammar

section.dia-slide per slide · roles dia-kicker / dia-title / dia-body / dia-caption · layout dia-columns / dia-stack / dia-figure · every color, size, and face reads a --dia-* token from one theme block.

behavior is data

data-dia-step orders reveals · scenes hold diagrams · no <script>, no on*= handlers in content. The runtime is embedded on save; a never-saved deck is still valid.

islands

What the dialect can't express is preserved verbatim — data-dia-island regions keep their scripts and stay alive. Verified conversion or verbatim preservation; never a silent maybe.

two validators, lockstep

src/model/validate.ts in the editor and a stdlib-only validate.py behind dia validate — identical rule ids, exit 1 on any error. The contract gate for humans and agents alike.

architecture · 03

One file, four surfaces.

  • 01edit — WYSIWYG over the live DOM; every gesture is one op in an append-only log.
  • 02present — the embedded runtime: arrows navigate, steps reveal. Works off file://.
  • 03import — foreign decks convert through a measured, human-reviewed pipeline.
  • 04agents — a copilot that proposes ops you preview; skills, AGENTS.md, MCP for every tool.

zero runtime dependencies in the document path — the editor bundle and deck runtime import nothing.

03 / 16diastīl
implementation · editor architecture · 04

A state hub, an op log, and one law: everything mutates through ops.

ops

An op is { label, author, apply(), invert() } — a closure over live elements, previous state captured at creation. Gestures batch to ONE undo step; authorship (you | copilot) reads honestly in the log.

isolation & round-trip

The deck lives in a shadow root — chrome styles can't leak in, deck tokens can't leak out. Editor artifacts strip on save; a saved deck re-serializes to identical bytes, so text-editor diffs stay clean.

04 / 16diastīl
implementation · scenes · 05

Scenes: author intent, derive geometry.

  • Nodes are data-x/y/w/h + a shape; labels center themselves; edge paths are derived artifacts — never hand-written.
  • Orthogonal routing is A* over node rects inflated 12px, bend-penalized, with a calm 1–2 bend fallback when boxed in.
  • Twelve shapes + freeform data-path (100×100-normalized) with center rotation; a point editor drags anchors and control points — one op per gesture.
  • Anything else — imported art, pen strokes — is a free element: draggable, restylable, promotable to a node at its bbox.

05 / 16diastīl
implementation · import · 06

Import runs the original, then measures everything.

execute

The source runs in a sandboxed iframe, history shimmed, fonts settled. A probe ladder learns its navigation — framework APIs → arrow keys → controls — and samples every slide in its activated state.

extract

Every element is stamped: geometry, computed typography, colors, animation delays, ::before markers materialized, shared defs inlined. Tokens are harvested from the majority ink and the clustered type scale.

convert

Measured facts carry. A gap-graph turns distances into fixed-or-elastic spacing; faces, weights, and colors bind to tokens; delays become steps; exact shapes lift deterministically into scenes.

implementation · the loss function · 07

The score: four axes, one ungameable multiplier.

score=(0.35D+0.30L+0.35A)⋅C3/2
displacementD=1−min⁡(da↔b,τ)/ττ = 4% of height
layoutL=2∑imin⁡(Ai,Bi)/∑i(Ai+Bi)16×9 ink cells
appearanceA=∑rwr(1−|Δc|r)blur r ∈ {1,3,8} px
completenessC=covered cells/inked cellsneighbor-tolerant

both renders rasterize through the same foreignObject pipeline at 384×216; ink masks binarize at ±16/channel against each raster's own estimated background. C multiplies, so deleting content can never be traded against polishing what remains.

a binary pixel match is a step function of displacement — text nudged 2px by font metrics scored ~0; the distance transform grades drift in proportion. separately, confidence = preserved text chars / total (text is sacred). score < 0.8 → repair rounds.

07 / 16diastīl
implementation · repair control flow · 08

Repair: a measurement-governed loop.

evidence = axis verdict ("this is an APPEARANCE problem — do not move content") · mismatch regions hit-tested against samples ("contains: the title") · measured deltas ("title: 64px vs 39px") · close-up crop pairs of the worst region · what the last rejected attempt did wrong. A repair that deletes source text is rejected outright; every candidate stays selectable — the human outranks the metric.

08 / 16diastīl
implementation · holding the line · 09

The ratchet, and untrusted output everywhere.

the corpus ratchet

Golden decks are captured with per-slide fidelity (and its axes) pinned in fixtures; FIDELITY_FLOORS fails the suite if a change re-captures lower. Tuning happens against pinned numbers — the ratchet has already caught two real regressions.

malformed output has a defined fate

Every model reply is recovered (fence-stripping, root extraction, op coercion), corrected (one round, shown its own bad reply), or refused with a reason. A crash never reaches the editor; dia eval measures the prompts themselves.

09 / 16diastīl
implementation · agents · 10

The copilot proposes; you preview; the log records.

  • One law: the model never edits directly. It emits typed ops through a tool call; the editor compiles them against the live DOM.
  • Proposals preview on the slide the moment they arrive — dashed frame, labeled — apply commits one batch; reject restores.
  • Targets resolve through a forgiving grammar: ids, "slide 3 title", css selectors, exact text. Unresolvable ops are marked with reasons and fed back for one automatic correction round.
  • Context is honest: the rail shows exactly what the model sees — slides, tokens, the current render, the imported original, and any regions you shade.
the service

A local FastAPI sidecar (127.0.0.1:8317) — skills are versioned markdown prompts over any OpenAI-compatible endpoint. No telemetry; the only outbound traffic is to the model you configured. The editor is complete without it.

every tool

Claude Code plugin · dia agents-md for AGENTS.md tools · dia mcp (stdlib stdio) for shell-less clients · dia validate as the acceptance test for all of them.

10 / 16diastīl
how to use it · 11

Five commands.

  1. dia new talk.html — a profile-valid starting deck
  2. dia talk.html — the editor; ⌘S writes back, disk changes reload
  3. dia ingest foreign.html — convert through review
  4. dia validate talk.html — the contract gate, exit 1 on errors
  5. dia present talk.html — it runs itself, off file://

the loop, for an agent:

scaffold → edit the html → dia validate → repeat. The validator is the real interface: an agent that satisfies it has produced a working deck, whatever produced it.

stdlib-only: validate and present need nothing installed beyond Python. Headless? every command takes --no-open.
