01 / 12diastil
the premise · 02

The file is the format.

A deck is a closed profile of HTML — small enough for a validator to hold forever.

grammar

section.dia-slide per slide · roles dia-kicker / dia-title / dia-body / dia-caption · layout dia-columns / dia-stack / dia-figure · every color, size, and face reads a --dia-* token from one theme block.

behavior is data

data-dia-step orders reveals · scenes hold diagrams · no <script>, no on*= handlers in content. The runtime is embedded on save; a never-saved deck is still valid.

islands

What the dialect can't express is preserved verbatim — data-dia-island regions keep their scripts and stay alive. Verified conversion or verbatim preservation; never a silent maybe.

two validators, lockstep

src/model/validate.ts in the editor and a stdlib-only validate.py behind dia validate — identical rule ids, exit 1 on any error. The contract gate for humans and agents alike.

architecture · 03

One file, four surfaces.

  • 01edit — WYSIWYG over the live DOM; every gesture is one op in an append-only log.
  • 02present — the embedded runtime: arrows navigate, steps reveal. Works off file://.
  • 03import — foreign decks convert through a measured, human-reviewed pipeline.
  • 04agents — a copilot that proposes ops you preview; skills, AGENTS.md, MCP for every tool.

zero runtime dependencies in the document path — the editor bundle and deck runtime import nothing.

03 / 12diastil
implementation · the op log · 04

Every edit is an op: apply, invert, replay.

the shape

An op is { label, author, apply(), invert() } — a closure over live elements that captures its previous state at creation. Gestures batch: a whole drag, a whole proposal, is ONE undo step.

byte-stable round-trip

A deck saved by diastil re-parses and re-serializes to identical bytes — held by tests as a hard guarantee. Text-editor diffs stay clean; the CLI's disk-watch can prove "editor is clean" by equality.

authorship

Ops carry author: you | copilot. Agent changes read honestly in the log, and a rejected proposal never touches it — previews apply outside the log and revert by running inverts in reverse.

editor artifacts

Selection rings, highlight layers, preview badges are .dia-editor-artifact nodes — serialization strips them, so chrome can never leak into the saved file.

04 / 12diastil
implementation · scenes · 05

Scenes: author intent, derive geometry.

  • Nodes are data-x/y/w/h + a shape; labels center themselves; edge paths are derived artifacts — never hand-written.
  • Orthogonal routing is A* over node rects inflated 12px, bend-penalized, with a calm 1–2 bend fallback when boxed in.
  • Twelve shapes + freeform data-path (100×100-normalized) with center rotation; a point editor drags anchors and control points — one op per gesture.
  • Anything else — imported art, pen strokes — is a free element: draggable, restylable, promotable to a node at its bbox.
05 / 12diastil
implementation · import · 06

Import runs the original, then measures everything.

execute — the source runs in a sandboxed iframe (history shimmed, fonts settled). A probe ladder learns its navigation: framework APIs → arrow keys → controls; every slide is sampled in its activated state.

extract — every element is stamped and sampled: geometry, computed typography, colors, animation delays, ::before markers materialized, shared SVG defs inlined. Tokens are harvested from the majority ink and clustered type scale.

convert — measured facts carry: a gap-graph turns distances into fixed-or-elastic spacing; faces, weights, colors bind to tokens; delays become steps; exact shapes lift deterministically into scenes.

06 / 12diastil
implementation · fidelity · 07

Fidelity is a measured composite, not a vibe.

Both renders rasterize through the same foreignObject pipeline at 384×216; ink masks binarize at a tight threshold. Four axes:

displacement

Symmetric truncated chamfer distance between ink masks (3-4 distance transform) — drift costs in proportion, saturating at 4% of height.

layout

Continuous Dice overlap of ink mass on a 16×9 grid — is the composition the same, at the scale a human judges first.

appearance

Mean color difference at blur radii 1/3/8px over the union mask — fine scale keeps rewrites expensive, coarse scales tolerate displacement.

completeness

Fraction of original ink cells still covered nearby, raised to the 1.5 power as a multiplier — dropping content can never win.

the explanation layer stays pixel-level: diff heatmaps, clustered mismatch regions hit-tested against the samples ("contains: the title"), and vertical-drift detection.

07 / 12diastil
implementation · the loop · 08

Repair is model-assisted, measurement-governed.

keep-best, gated

Under-threshold slides get repair rounds (cap 3). A candidate that re-measures lower — or that deletes source text — is rejected outright. Every candidate stays selectable in a versions row; the human outranks the metric.

the prompt is the evaluation

The model receives the axis scores ("this is an APPEARANCE problem"), named regions, measured element deltas ("title: 64px vs 39px"), close-up crops of the worst region, and what its last rejected attempt did wrong.

the corpus ratchet

Golden decks are captured with per-slide fidelity pinned in fixtures; FIDELITY_FLOORS fails the suite if a change re-captures lower. Tuning happens against pinned numbers, never live hand-measurement.

untrusted output, everywhere

Model replies are recovered (fence-stripping, root-element extraction, op coercion), corrected (one round, shown its own bad reply), or refused with a reason — malformed output can never take the editor down.

08 / 12diastil
implementation · agents · 09

The copilot proposes; you preview; the log records.

  • One law: the model never edits directly. It emits typed ops through a tool call; the editor compiles them against the live DOM.
  • Proposals preview on the slide the moment they arrive — dashed frame, labeled — apply commits one batch; reject restores.
  • Targets resolve through a forgiving grammar: ids, "slide 3 title", css selectors, exact text. Unresolvable ops are marked with reasons and fed back for one automatic correction round.
  • Context is honest: the rail shows exactly what the model sees — slides, tokens, the current render, the imported original, and any regions you shade.
the service

A local FastAPI sidecar (127.0.0.1:8317) — skills are versioned markdown prompts over any OpenAI-compatible endpoint. No telemetry; the only outbound traffic is to the model you configured. The editor is complete without it.

every tool

Claude Code plugin · dia agents-md for AGENTS.md tools · dia mcp (stdlib stdio) for shell-less clients · dia validate as the acceptance test for all of them.

09 / 12diastil
how to use it · 10

Five commands.

  1. dia new talk.html — a profile-valid starting deck
  2. dia talk.html — the editor; ⌘S writes back, disk changes reload
  3. dia ingest foreign.html — convert through review
  4. dia validate talk.html — the contract gate, exit 1 on errors
  5. dia present talk.html — it runs itself, off file://

the loop, for an agent:

scaffold → edit the html → dia validate → repeat. The validator is the real interface: an agent that satisfies it has produced a working deck, whatever produced it.

stdlib-only: validate and present need nothing installed beyond Python. Headless? every command takes --no-open.

the numbers · 11

Held by tests, not intentions.

200+ tests: round-trip byte-stability, both validators, the metric's ranking properties, the marker ladder, preview semantics.

corpus

Golden decks with pinned per-slide fidelity + confidence floors — re-captured deliberately, diffed in review, never drive-by.

profile

One contract (PROFILE.md), two implementations in lockstep, identical rule ids — the editor and the CLI can never disagree.

evals

dia eval scores the model skills against golden cases — prompt and endpoint changes are measured, like everything else.
